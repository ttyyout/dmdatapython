# Cursor Rules (Goal-Aware, Non-Intrusive)

> ⚠️ 이 Rules는 **행동 제약과 맥락 유지용**이며,
> **코드/설계/결과물의 내용에는 직접적인 영향을 주지 않는다.**
> 사용자가 목표를 명시하지 않은 경우, 아래 목표는 **완전히 무시**한다.

---

## 0. 최상위 원칙 (가장 중요)

- **사용자가 명시적으로 목표를 언급한 경우에만** 이 Rules의 목표 섹션을 고려한다.
- 사용자가 목표를 언급하지 않은 요청에서는:
  - 목표, UI, 지진, OBS, TTS 등 모든 도메인 맥락을 **추론하거나 끼워 맞추지 않는다.**
- 본 Rules는:
  - 코드 품질
  - 오류 방지
  - 구조적 일관성
  을 위한 **메타 규칙**이며, 기능 강제나 설계 변경을 유도하지 않는다.

---

## 1. 공통 코드 규칙

- 코드를 수정하라는 요청이 있을 경우:
  - **컴파일/런타임 오류가 발생하지 않도록** 한다.
  - 추측 기반 수정 금지
  - 누락된 맥락이 있으면 명확히 질문한다.
- 기존 코드의 의미를 바꾸는 수정은:
  - 사용자가 명시적으로 요구한 경우에만 허용한다.
- 예시는 **설명용**으로만 제공하며:
  - 실제 구현 강제 ❌
  - 구조 강제 ❌

---

## 2. 목표 정의 (사용자가 목표를 말한 경우에만 적용)

### 2.1 궁극적 목표 (아직 달성되지 않음)

- UI 표시가 가능한 앱을 개발하는 것을 목표로 한다.
- 현재는 **목표 정의 단계**이며, 완성 상태를 전제하지 않는다.

---

## 3. UI 구조 목표 (설명용, 강제 아님)

> 아래 항목은 **사용자가 제시한 목표 설명을 그대로 보존**하기 위한 섹션이다.
> 이 내용은 설계 예시이며, 자동 적용되지 않는다.

### 3.1 창 구성

- 앱 실행 시 창이 **2개** 표시됨

#### 창 A: 방송용 정보 창
- 방송에 필요한 정보만 표시
- 윈도우 작업 표시줄 정도의 크기

#### 창 B: 상세 정보 창
- 모든 세부 정보 표시

---

### 3.2 상세 정보 창 상단: 상황판

- 현재 지진 상황을 표시하는 "상황판" 존재
- 표시 예시:
  - 긴급지진속보
  - 진도속보
- 각 상황에 대해:
  - 화살표
  - 점멸 표시
  등 시각적 강조 가능

- 지진이 여러 개인 경우:
  - 각 지진을 **구분하여 동시에 표시 가능**해야 함

---

### 3.3 상황판 하단: 탭 구조

#### 3.3.1 메인 탭

- 현재 발생 중인 지진 정보 표시
- 상황에 따라 표시 내용 변경:
  - 긴급지진속보 → 긴급지진속보 정보
  - 진도속보 → 진도속보 정보

- UI 참고 예시:
  - **JQuake 좌상단 정보 박스와 유사한 형태**

- 현재 지진이 2개 이상일 경우:
  - 메인 탭 내부에 **추가 탭**을 두어 지진별 선택 가능

- 메인 탭 하단:
  - 최근 발생한 지진들의 **지진 목록** 표시

---

#### 3.3.2 OBS 탭

- ASS 및 OBS WebSocket 사용
- 방송 관리자 성격의 기능
- 표시 정보 예시:
  - 현재 OBS 장면
  - 현재 상황에서 수행 중인 동작

---

#### 3.3.3 TTS 탭

- 현재 발표된 정보를 음성으로 읽기
- 예시 문장 (설명용):
  - "(진원지)에서 지진이 발생했습니다. 규모는 (규모)입니다."

- TTS 문구는:
  - 명령어처럼 관리 가능해야 함

## 5. 예시 섹션의 위치에 대한 규칙

- 본 문서 내의 "예시", "표시 예시", "UI 참고" 항목은:
  - 이해를 돕기 위한 설명용 예시이다.
  - 실제 구현을 강제하지 않는다.
- 예시가 있다는 이유로:
  - 해당 UI/기능이 이미 존재한다고 가정하지 않는다.

---

## 6. 요약 (AI 내부 기준)

- 목표는 **존재하지만 자동 적용되지 않는다**
- 사용자가 목표를 언급한 경우에만 참조한다
- 예시는 **예시로만 취급**한다
- 제공된 정보는 **누락 없이 유지**한다
- 오류 없는 코드 수정이 최우선이다
